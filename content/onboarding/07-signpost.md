## That's the tour

In about fifteen minutes you captured an intent, had it built, and watched
a memoryless agent answer from the project's own record. That round trip —
spec to commit to code, readable by any agent — is the core of AIDA, and
you now have the one idea the rest of it builds on. When you want the
guided depth — the distributed store, search, the full command surface —
the 35-exercise track is one command away: `aida-tutor list`.

## What's next — Act II, when it settles

There is a second half to AIDA: a collaboration layer where several agents
and people hand work back and forth — a shared queue, named roles, scoped
sessions, an orchestrator that drives reviews and merges. It is powerful
and still moving fast, so this tour deliberately stops short of it — when
that surface settles, it becomes Act II of this tutor. Until then, the
round trip you just did is enough to put AIDA to work on real code today:
capture intent, trace it to code, let the project remember.
